1. Общее описание проекта
Интернет-магазин «Lume» — это современная платформа для онлайн-покупки стильной одежды, ориентированная на пользователей, ценящих качество, дизайн и индивидуальность. Название «Lume» отражает концепцию «света» — яркости, уникальности и уверенности в себе через одежду.

2. Цели проекта
Создать эстетичный и удобный интерфейс для просмотра и выбора товаров.
Реализовать MVP-функционал: навигация, просмотр товаров, добавление в корзину, мобильное меню.
Обеспечить полную адаптивность под все устройства.
Подготовить структуру для последующего подключения бэкенда (Django).
3. Целевая аудитория
Женщины и мужчины 18–45 лет.
Пользователи, интересующиеся модой, качеством материалов и современным дизайном.
Люди, ищущие одежду для разных случаев: офис, вечеринки, прогулки, праздники.

5. Функциональные требования (MVP — детализированный)
 Важно: Все функции на текущем этапе реализуются на клиентской стороне (JavaScript + localStorage). Бэкенд — на следующих этапах.

 Шапка (Header)

Меню: Главная, Товары, Категории, О нас, Контакты

Поведение:
Элемент
Описание
Логотип
Ссылка на главную страницу (#home). При клике — плавная прокрутка к началу страницы.
Меню (Desktop)
Статичное горизонтальное меню. При наведении — подчёркивание золотым цветом. При клике — плавная прокрутка к соответствующему разделу (якорь).
Мобильное меню (Burger)
Отображается при ширине экрана < 768px. При клике — открывает/закрывает боковое меню. Закрывается при клике на любой пункт меню или вне меню.
Иконка Поиска
При клике — открывается prompt с текстом: “Что вы ищете?”. После ввода — alert с сообщением: “Поиск товаров по запросу: “[введённый текст]” (в реальном магазине здесь будет переход на страницу поиска)”.
Иконка Профиля
При клике — alert: “Вход в личный кабинет\n(В реальном магазине здесь будет форма входа)”.
Иконка Корзины
При клике — если cartCount > 0: alert “В вашей корзине X товар(ов)\n(В реальном магазине здесь будет переход на страницу корзины)”. Если cartCount = 0: alert “Ваша корзина пуста”.

 Блок «Популярные товары» (Products Grid)
Элементы:
Заголовок: “Популярные товары”
Сетка карточек товаров (4 шт. в десктопе, 1 в мобильной версии)
Каждая карточка содержит:
Изображение
Категория (тег)
Название товара
Цена (с возможным старым значением)
Кнопка “В корзину”
Поведение:
Элемент
Описание
Карточка товара
При наведении — тень увеличивается, изображение масштабируется на 1.05.
Кнопка “В корзину”
При клике —
1. Увеличивает счётчик корзины (cartCount++).
2. Обновляет визуальный счётчик на иконке корзины.
3. Выводит alert: “Товар “[название]” добавлен в корзину! В корзине теперь X товар(ов)”.
4. Не перезагружает страницу, не отправляет данные на сервер.
Цена со скидкой
Если есть старая цена — она зачёркнута, новая — жирная.

6. Интерактивные элементы (детализация)
 Мобильное меню
Показывается при ширине экрана ≤ 768px.
Кнопка “бургер” — переключает класс active у <nav>, что вызывает анимацию слева направо.
Закрытие — при клике на любой пункт меню или вне меню (не на самой навигации).
 Плавная прокрутка (Smooth Scroll)
Работает для всех ссылок с href^="#".
При клике — прокручивает к нужному элементу с отступом 80px сверху (для того чтобы не закрывалось меню).
 Счётчик корзины
Инициализируется при загрузке страницы (cartCount = 0).
Счётчик — это <span> с классом cart-counter, позиционированный абсолютно относительно иконки корзины.
При добавлении товара — cartCount++, updateCartCounter() обновляет значение и показывает/скрывает счётчик.
Визуально: цифра в круглом золотом бейдже.
 Уведомления (Alerts)
Используются для демонстрации функционала.
Не должны мешать пользователю — после закрытия не блокируют интерфейс.
7. Нефункциональные требования
Адаптивность:
Десктоп (≥ 992px): полная ширина, сетка 4 колонки.
Планшет (768–991px): сетка 2–3 колонки, меню скрыто.
Мобильный (≤ 767px): сетка 1 колонка, меню — бургер, шрифты уменьшены.
Производительность:
Изображения оптимизированы (можно использовать WebP в будущем).
CSS/JS минифицированы (на продакшене).
Доступность (a11y):
Все изображения имеют alt.
Семантические теги (header, nav, section, footer).
Кнопки и ссылки — доступны с клавиатуры.
SEO:
Заголовки H1–H6 используются правильно.
Метатеги (title, description) — будут добавлены на следующем этапе.
